<!-- Cart drawer talks to Shopify Ajax Cart API. Key endpoints: -->

POST /cart/change.js
  body: { id: <variant_id>, quantity: <new_qty> }
  returns: updated cart JSON { item_count, total_price (in cents), items: [...] }

GET /cart.js
  returns: current cart JSON (same shape as above)

Cart item shape (from Shopify):
  item.id           — variant ID (used as the line item key)
  item.title        — product title
  item.variant_title — variant label (e.g. "S / Red")
  item.image        — thumbnail URL
  item.quantity     — current quantity
  item.final_price  — price in cents (divide by 100 for display)
  item.final_line_price — line total in cents

cart.total_price — cart subtotal in cents
cart.item_count  — total item count (sum of quantities)

| cart-drawer.js → /cart/change.js | Shopify Ajax Cart API; session-scoped, no auth token needed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05 | Tampering | renderCart innerHTML | mitigate | Cart item data originates from Shopify's own API response (not user input). Product titles and variant titles are Shopify-managed. Do not use `eval` or `innerHTML` on user-typed strings. |
| T-02-06 | Tampering | quantity controls | mitigate | Enforce `Math.max(1, qty - 1)` client-side; Shopify validates server-side — double defense |
| T-02-07 | Elevation of Privilege | checkout link | accept | Link uses `{{ routes.cart_url }}/checkout` — Shopify's own checkout URL; no custom auth required |
| T-02-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages in this plan |
</threat_model>
