<objective>
Load the UpPromote tracking pixel on every page and wire referral code capture from URL
params through to checkout. Referral param capture is handled by `assets/ref-capture.js`
(created in plan 06-03) — upromote.js reads shared sessionStorage state rather than
re-reading URL params independently. Commission structure (Standard 10% / Silver 12% /
Gold 15%, Ambassador, Customer Referral 5%) and fraud rules (self-referral block, IP
clustering, 14-day hold) are configured in the UpPromote dashboard — no theme code required.

Purpose: Without the tracking pixel and referral code pass-through, UpPromote cannot
attribute sales to affiliate links and discount codes will not apply at checkout.

Output: Pixel loads on every page; `?ref=CODE` captured to sessionStorage via ref-capture.js;
checkout URL includes the discount code so Shopify applies it automatically.
</objective>

- [ ] **Task 1: Inject UpPromote tracking pixel in theme.liquid**

  Files: `layout/theme.liquid`

  The UpPromote tracking pixel is a `<script>` tag that must load on every page, including
  the order confirmation page (for conversion attribution). Add in `<head>` after the GA4
  block:

  ```liquid
  {%- if settings.upromote_merchant_id != blank -%}
    <script async
      src="https://cdn.upromote.me/js/affiliate.min.js"
      data-merchant-id="{{ settings.upromote_merchant_id }}">
    </script>
  {%- endif -%}
  ```

  The `upromote_merchant_id` setting was already declared in `config/settings_schema.json`
  in plan 06-01. No additional schema changes needed.

  Also add the module script load before `</body>`. Note: `ref-capture.js` must load before
  `upromote.js` so sessionStorage is populated when upromote.js reads it. Because both are
  `type="module"` the browser defers them; load order in HTML determines execution order:

  ```liquid
  {%- if settings.upromote_merchant_id != blank -%}
    <script type="module" src="{{ 'ref-capture.js' | asset_url }}"></script>
    <script type="module" src="{{ 'upromote.js' | asset_url }}"></script>
  {%- endif -%}
  ```

  If plan 06-03's GA4 block already emits the `ref-capture.js` module tag, do NOT add a
  second `ref-capture.js` tag here — a single tag is sufficient for both modules because
  ES module scripts are deduplicated by the browser. Check whether the tag already exists
  before adding.

  Verify: `grep -c "upromote_merchant_id" layout/theme.liquid` returns >= 1.

- [ ] **Task 2: Create assets/upromote.js — referral capture + checkout wiring**

  Files: `assets/upromote.js`, `assets/cart-drawer.js`

  `assets/upromote.js`:

  Import `getInfluencerCode` from `./ref-capture.js`. Do NOT read `location.search` or
  `sessionStorage` directly for the ref param — all referral state is owned by ref-capture.js.

  On DOMContentLoaded:
  1. Call `getInfluencerCode()` (imported from `./ref-capture.js`).
     This returns `sessionStorage.getItem('upromote_ref')` if set, else falls back to
     `utm_campaign`. ref-capture.js has already written `upromote_ref` to sessionStorage
     at module evaluation time (before DOMContentLoaded), so the value is available.
  2. If code is non-null, also store as `discount_code` in sessionStorage (same value —
     UpPromote uses the affiliate's referral code as the Shopify discount code).

  Checkout URL wiring:
  - Query `document.querySelectorAll('a[href*="/checkout"]')` on DOMContentLoaded.
  - For each link, if `sessionStorage.getItem('discount_code')` is set, append
     `?discount=CODE` to the href (or `&discount=CODE` if query string already present).
  - Also dispatch a custom event `upromote:ref-captured` with `{ detail: { code } }` so
    cart-drawer.js can react if needed.

  In `assets/cart-drawer.js`:
  - Listen for `upromote:ref-captured` event on `document`.
  - When the checkout button's href is set (it is built dynamically from `/cart.js`
    response), append `?discount=${event.detail.code}` to the checkout URL.
  - This covers the case where cart-drawer renders after upromote.js fires.

  Commission tiers and fraud rules are fully dashboard-side — document in `06-CONTEXT.md`
  that the merchant must configure:
  - 3-tier commission: Standard 10%, Silver 12%, Gold 15%
  - Ambassador program
  - Customer Referral 5%
  - Self-referral block: enabled in UpPromote Settings → Fraud Detection
  - IP clustering threshold: set to 3 conversions per IP per 24h
  - Conversion hold: 14 days

  Verify: `assets/upromote.js` exists; `grep -c "getInfluencerCode" assets/upromote.js` >= 1;
  `grep -c "upromote:ref-captured" assets/cart-drawer.js` >= 1.

|----------|-------------|
| URL param → sessionStorage | Referral code written to sessionStorage exclusively by ref-capture.js; upromote.js reads only via getInfluencerCode() |
| sessionStorage → checkout URL | Discount code appended to checkout href |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-09 | Tampering | ?ref= param manipulation | mitigate | Code passed as Shopify discount param; Shopify validates discount codes server-side; invalid codes are silently ignored at checkout |
| T-06-10 | Spoofing | Affiliate self-referral | mitigate | Self-referral block configured in UpPromote dashboard (not theme-side) |
| T-06-11 | Elevation of Privilege | Stacked discount codes | accept | Shopify limits discount stacking; UpPromote's code replaces any existing discount param |
| T-06-SC | Tampering | No npm installs this plan | accept | Pure vanilla JS |
</threat_model>
