- [ ] **Task 2: End-to-end verification checklist (human checkpoint)**

  Type: `checkpoint:human-verify`

  With all plans 06-01 through 06-05 executed and placeholder IDs in settings_data.json,
  run `shopify theme dev` and verify:

  **Klaviyo:**
  1. Open DevTools → Network → filter "klaviyo". Confirm `klaviyo.js?company_id=` script
     loads in `<head>` (will 404 with placeholder ID — that is expected; confirm tag is
     present).
  2. Navigate to `/account` as a logged-in customer. In Console, run
     `window.klaviyo?.identify` — should not throw (method exists on window).
  3. Visit a product page. Confirm `data-bis-variant` attribute is present on OOS variant
     back-in-stock form.

  **GA4:**
  1. DevTools → Network → filter "gtag". Confirm `gtag.js?id=G-XXXXXXXXXX` tag in `<head>`.
  2. In Console, run `window.dataLayer` — should be an array with at least one config push.
  3. Add a product to cart. In Console, run `window.dataLayer` — should contain an
     `add_to_cart` event entry.
  4. Open cart drawer and click Checkout. Confirm `begin_checkout` event in dataLayer.
  5. Navigate to a collection page. Confirm `view_item_list` event in dataLayer.

  **UpPromote:**
  1. Visit `/?ref=TESTCODE` in a new tab. Open Console → `sessionStorage.getItem('upromote_ref')`
     — should return `'TESTCODE'`.
  2. Open cart drawer. Inspect the Checkout button href — should contain `?discount=TESTCODE`.

  **Cloudinary:**
  1. On any collection page, right-click a product card image → Copy image address.
  2. Confirm URL contains `res.cloudinary.com/.../image/fetch/f_auto,q_auto` (will 404
     with placeholder cloud name — confirm URL structure is correct).
  3. In `config/settings_data.json`, temporarily set `cloudinary_cloud_name` to blank
     and reload — confirm product card images fall back to Shopify CDN URLs.

  Resume signal: Type "verified" to continue, or describe any failures.

- [ ] **Task 3: Update 06-CONTEXT.md with GA4 purchase snippet + final integration notes**

  Files: `.planning/phases/06-integrations/06-CONTEXT.md`

  Append to `06-CONTEXT.md`:

  **GA4 purchase event (Shopify Additional Scripts)**

  This snippet must be added by the owner in Shopify Admin → Settings → Checkout →
  Additional Scripts. It fires only on the order status page:

  ```liquid
  {% if first_time_accessed %}
  <script>
    window.dataLayer = window.dataLayer || [];
    function gtag(){dataLayer.push(arguments);}
    gtag('event', 'purchase', {
      transaction_id: '{{ order.order_number }}',
      affiliation: '{{ shop.name }}',
      value: {{ checkout.total_price | money_without_currency }},
      tax: {{ checkout.tax_price | money_without_currency }},
      shipping: {{ checkout.shipping_price | money_without_currency }},
      currency: 'USD',
      items: [
        {% for line_item in checkout.line_items %}
        {
          item_id: '{{ line_item.variant_id }}',
          item_name: '{{ line_item.title | escape }}',
          item_category: '{{ line_item.product.type | escape }}',
          price: {{ line_item.price | money_without_currency }},
          quantity: {{ line_item.quantity }}
        }{% unless forloop.last %},{% endunless %}
        {% endfor %}
      ]
    });
  </script>
  {% endif %}
  ```

  Also document Klaviyo post-purchase, welcome series, win-back, and VIP early access flow
  configurations (Klaviyo dashboard only, no additional theme code required).

  Verify: `grep -c "purchase" .planning/phases/06-integrations/06-CONTEXT.md` >= 1.
