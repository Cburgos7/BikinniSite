<objective>
Implement Cloudinary image optimization via the fetch-URL pattern. All product images are
served through Cloudinary's CDN using automatic format selection (f_auto) and quality
optimization (q_auto) over Shopify CDN source URLs. Pure Liquid — no JS, no build step.

Purpose: Reduces image payload by 30–60% via WebP/AVIF auto-format and perceptual quality
compression, improving LCP scores critical for conversion on mobile.

Output: `snippets/cloudinary-img.liquid` wraps any Shopify img_url output; product cards
and PDP gallery use it; fallback to raw Shopify CDN when cloud name is not configured.
</objective>

<tasks>

- [ ] **Task 1: Create snippets/cloudinary-img.liquid**

  Files: `snippets/cloudinary-img.liquid`

  This snippet accepts these variables (passed via `render` tag):
  - `image` — Shopify image object (required)
  - `width` — integer, output width in px (required, e.g. 600)
  - `alt` — string, alt text (required)
  - `class` — string, Tailwind classes (optional, default empty)
  - `loading` — string, "lazy" or "eager" (optional, default "lazy")

  Snippet logic:

  ```liquid
  {%- assign shopify_url = image | img_url: width | prepend: 'https:' -%}
  {%- if settings.cloudinary_cloud_name != blank -%}
    {%- assign src = 'https://res.cloudinary.com/'
      | append: settings.cloudinary_cloud_name
      | append: '/image/fetch/f_auto,q_auto,w_'
      | append: width
      | append: '/'
      | append: shopify_url -%}
  {%- else -%}
    {%- assign src = shopify_url -%}
  {%- endif -%}
  <img
    src="{{ src }}"
    alt="{{ alt | escape }}"
    width="{{ width }}"
    loading="{{ loading | default: 'lazy' }}"
    {%- if class != blank %} class="{{ class }}"{% endif %}
  >
  ```

  Note: `img_url` with a plain integer produces `_{width}x` Shopify CDN URLs. Use the
  Liquid pattern `image | img_url: width` where width is the number without a unit string.
  If the theme uses named size strings (e.g., `'600x'`), adjust accordingly to match the
  existing convention in `snippets/product-card.liquid`.

  Verify: `test -f snippets/cloudinary-img.liquid` and snippet contains
  `res.cloudinary.com` string.

- [ ] **Task 2: Apply cloudinary-img snippet to product cards and PDP gallery**

  Files: `snippets/product-card.liquid`, `sections/product-main.liquid`

  In `snippets/product-card.liquid`, locate the product card image tag (currently uses
  `img_url` directly or an `<img>` tag). Replace with:

  ```liquid
  {%- render 'cloudinary-img',
    image: card_product.featured_image,
    width: 600,
    alt: card_product.featured_image.alt | default: card_product.title,
    class: 'w-full h-full object-cover',
    loading: 'lazy'
  -%}
  ```

  In `sections/product-main.liquid`, locate the main gallery image(s) — the primary image
  and any thumbnail images in the gallery loop. Replace each `<img>` with a `render`
  call to `cloudinary-img`. Use `width: 1200` for the main gallery image and `width: 150`
  for thumbnails. Set `loading: 'eager'` on the first/primary image (above the fold),
  `loading: 'lazy'` on all others.

  Do not change surrounding markup, classes, or data attributes — only replace the `<img>`
  element itself with the snippet render call.

  Verify: `grep -c "cloudinary-img" snippets/product-card.liquid` >= 1;
  `grep -c "cloudinary-img" sections/product-main.liquid` >= 1.
