<objective>
Wire Klaviyo behavioral tracking events for abandoned cart and back-in-stock into the
existing cart and PDP JS modules. Welcome series, post-purchase, win-back, and VIP early
access flows require only Klaviyo dashboard configuration — no additional theme code.

Purpose: Klaviyo abandoned cart flow requires an "Added to Cart" track event; without it
the flow cannot identify who abandoned. Back-in-stock requires a subscribe-to-list call
for out-of-stock variant visitors.

Output: `assets/klaviyo-flows.js` provides track helpers; `assets/pdp.js` calls them after
add-to-cart; a back-in-stock snippet renders on PDP for OOS variants.
</objective>

<tasks>

- [ ] **Task 1: Create assets/klaviyo-flows.js — track helpers**

  Files: `assets/klaviyo-flows.js`

  This ES module exports two functions:

  `trackAddedToCart(cartItem)` — calls `window.klaviyo?.track("Added to Cart", {...})` with
  properties: `ProductName`, `ProductID` (variant id), `SKU`, `Categories`, `ImageURL`,
  `URL` (product url), `Brand`, `Price`, `CompareAtPrice`, `Quantity`.

  Map from the Shopify `/cart/add.js` response object that `pdp.js` already receives. The
  `ImageURL` field: use the `featured_image` URL from the response if present.

  `subscribeBackInStock(email, variantId, platform)` — POSTs to Klaviyo's back-in-stock
  endpoint:
  ```
  POST https://a.klaviyo.com/client/back-in-stock-subscriptions/
  Headers: { 'content-type': 'application/json', 'revision': '2023-12-15' }
  Body: {
    data: {
      type: 'back-in-stock-subscription',
      attributes: { channels: ['EMAIL'], profile: { data: { type: 'profile', attributes: { email } } } },
      relationships: { variant: { data: { type: 'catalog-variant', id: `$shopify:::$default:::${variantId}` } } }
    }
  }
  ```
  The Klaviyo company ID for the client API auth header `Authorization: Klaviyo-API-Key ...`
  is read from `document.body.dataset.klaviyoCompanyId` (add this data attribute to
  `<body>` in `layout/theme.liquid` alongside the existing customer data attributes).

  Verify: `assets/klaviyo-flows.js` exists and contains `trackAddedToCart` and
  `subscribeBackInStock` exports.

- [ ] **Task 2: Wire pdp.js + add back-in-stock form to PDP section**

  Files: `assets/pdp.js`, `sections/product-main.liquid` (or equivalent PDP section)

  In `assets/pdp.js`, after the successful `/cart/add.js` fetch response in `addToCart()`:
  - Import `trackAddedToCart` from `./klaviyo-flows.js`
  - Call `trackAddedToCart(responseJson)` — `responseJson` is the Shopify cart item returned
    by the add-to-cart endpoint

  In the PDP section Liquid file, add a back-in-stock form block rendered only when the
  selected variant is unavailable (`variant.available == false`). Use a `<div>` with
  `data-bis-variant="{{ variant.id }}"` and an email `<input>` + submit button. The form
  submit is handled in `klaviyo-flows.js`: attach a `submit` event listener to
  `[data-bis-variant]` forms on `DOMContentLoaded`, call `subscribeBackInStock()`, show
  inline success/error message.

  Do not modify the add-to-cart button logic beyond the track call — pdp.js already handles
  variant selection, quantity, and cart drawer trigger.

  Verify: `assets/pdp.js` contains `trackAddedToCart` import; PDP section contains
  `data-bis-variant` attribute string.
