<objective>
Embed the Klaviyo JS snippet into the theme, create reusable signup form sections with
TCPA-compliant SMS opt-in language, and wire customer identification on account pages.

Purpose: Foundation for all Klaviyo flows — abandoned cart, welcome series, post-purchase,
win-back, VIP early access, and back-in-stock all require the JS snippet and identify hook.

Output: Klaviyo loads on every page; signup forms are embeddable via theme editor; SMS opt-in
is TCPA-compliant (explicit checkbox, not pre-checked, with required consent language).
The TCPA SMS checkbox is rendered by Klaviyo's form builder when SMS is enabled on the form
in the Klaviyo dashboard — no custom theme code handles the checkbox. Checkout SMS compliance
(opt-in at checkout) is configured in Klaviyo's checkout integration settings, not via
Shopify Additional Scripts in the theme.
</objective>

<tasks>

- [ ] **Task 1: Inject Klaviyo JS snippet into theme.liquid**

  Files: `layout/theme.liquid`

  In the `<head>` block, after `{{ content_for_header }}`, add:

  ```liquid
  {%- if settings.klaviyo_company_id != blank -%}
    <script async type="text/javascript"
      src="https://static.klaviyo.com/onsite/js/klaviyo.js?company_id={{ settings.klaviyo_company_id }}">
    </script>
  {%- endif -%}
  ```

  The `klaviyo_company_id` setting is declared in `config/settings_schema.json` under a new
  "Integrations" section group (add if not present):

  ```json
  {
    "name": "Integrations",
    "settings": [
      { "type": "text", "id": "klaviyo_company_id", "label": "Klaviyo Company ID", "info": "Found in Klaviyo → Account → API Keys" },
      { "type": "text", "id": "ga4_measurement_id",  "label": "GA4 Measurement ID",  "info": "Format: G-XXXXXXXXXX" },
      { "type": "text", "id": "cloudinary_cloud_name","label": "Cloudinary Cloud Name","info": "Your Cloudinary account cloud name" },
      { "type": "text", "id": "upromote_merchant_id", "label": "UpPromote Merchant ID","info": "Found in UpPromote dashboard → Settings" }
    ]
  }
  ```

  Add all four integration settings in one commit — downstream plans reference the same
  settings group.

  Verify: `grep -c "klaviyo_company_id" config/settings_schema.json` returns > 0.

- [ ] **Task 2: Create sections/klaviyo-forms.liquid**

  Files: `sections/klaviyo-forms.liquid`

  This section renders an inline Klaviyo form embed. It accepts a `form_id` schema setting
  (text) so the merchant can paste their Klaviyo form ID via the theme editor.

  The section output:

  ```liquid
  {%- if section.settings.form_id != blank -%}
    <div class="klaviyo-form-{{ section.settings.form_id }}"></div>
  {%- endif -%}
  ```

  TCPA SMS opt-in note: Klaviyo's own form builder handles the SMS checkbox rendering when
  SMS is enabled on the form in the Klaviyo dashboard. The `{% schema %}` block must include:
  - `form_id` (text) — Klaviyo form ID
  - `section_heading` (text, default "Join the Club") — optional heading above form
  - `show_heading` (checkbox, default true)

  Add `{% schema %}` with `name: "Klaviyo Signup Form"` and `presets` so it appears in the
  theme editor's "Add section" menu.

  Verify: section file exists and contains `klaviyo-form-` div and `{% schema %}`.

- [ ] **Task 3: Create assets/klaviyo.js — identify + track helpers**

  Files: `assets/klaviyo.js`

  This ES module runs on pages where `customer` is available. Because Liquid cannot directly
  inject into an ES module, use a data attribute on `<body>` to pass customer data.

  In `layout/theme.liquid` `<body>` tag, add data attributes:

  ```liquid
  <body
    class="bg-cream text-deep font-body pt-16"
    data-customer-email="{{ customer.email | escape }}"
    data-customer-id="{{ customer.id }}"
  >
  ```

  `assets/klaviyo.js` reads these attributes and calls `klaviyo.identify()`:

  ```js
  // Do not inline — this is the intent, not the literal file content.
  // On DOMContentLoaded: read data-customer-email from body.
  // If email is non-empty and window.klaviyo exists, call:
  //   window.klaviyo.identify({ '$email': email, '$id': customerId })
  // Export a trackEvent(eventName, properties) helper that wraps
  //   window.klaviyo?.track(eventName, properties) — used by GA4 and cart plans.
  ```

  Load the script as a module in `layout/theme.liquid` before `</body>`:

  ```liquid
  {%- if settings.klaviyo_company_id != blank -%}
    <script type="module" src="{{ 'klaviyo.js' | asset_url }}"></script>
  {%- endif -%}
  ```

  Verify: `assets/klaviyo.js` exists; `layout/theme.liquid` contains `klaviyo.js` asset_url reference.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Tampering | `data-customer-email` on body | mitigate | Value is escaped via Liquid `escape` filter; read-only attribute; no eval path |
| T-06-02 | Information Disclosure | Klaviyo identify() exposes email in JS | accept | Standard Klaviyo pattern; email already known to browser session |
| T-06-03 | Spoofing | Klaviyo company ID in settings | accept | Settings admin-gated; wrong ID simply fails to load forms |
| T-06-SC | Tampering | No npm installs this plan | accept | Pure Liquid + vanilla JS, no package manager invocations |
</threat_model>
