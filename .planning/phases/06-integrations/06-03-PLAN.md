<objective>
Implement GA4 enhanced e-commerce tracking: gtag.js in `<head>`, event firing from
existing JS modules (pdp.js, cart-drawer.js), and collection-level list events from a
new ga4.js module. Wire the custom `influencer_code` dimension from UTM params.

Purpose: Without GA4 enhanced e-commerce, the store has no visibility into browse-to-
purchase funnel drop-off, which is critical for a drops-model business.

Output: All six required events fire; `influencer_code` custom dimension propagates on
every hit; `purchase` event fires on order status page via Shopify Additional Scripts.
</objective>

- [ ] **Task 1: Inject gtag.js and data islands into theme.liquid**

  Files: `layout/theme.liquid`

  In `<head>`, after the Klaviyo snippet block, add:

  ```liquid
  {%- if settings.ga4_measurement_id != blank -%}
    <script async src="https://www.googletagmanager.com/gtag/js?id={{ settings.ga4_measurement_id }}"></script>
    <script>
      window.dataLayer = window.dataLayer || [];
      function gtag(){dataLayer.push(arguments);}
      gtag('js', new Date());
      gtag('config', '{{ settings.ga4_measurement_id }}', { send_page_view: true });
    </script>
  {%- endif -%}
  ```

  After the gtag block, inject a Liquid data island so ga4.js can read page context
  without additional fetch calls:

  ```liquid
  <script id="ga4-page-data" type="application/json">
    {
      "pageType": {{ request.page_type | json }},
      "currency": "USD",
      {%- if product -%}
      "product": {
        "id": {{ product.id | json }},
        "name": {{ product.title | json }},
        "brand": {{ shop.name | json }},
        "category": {{ product.type | json }},
        "variant": {{ product.selected_or_first_available_variant.id | json }},
        "price": {{ product.selected_or_first_available_variant.price | money_without_currency | json }},
        "sku": {{ product.selected_or_first_available_variant.sku | json }},
        "imageUrl": {{ product.featured_image | img_url: '600x' | json }}
      },
      {%- endif -%}
      {%- if collection -%}
      "collection": {
        "id": {{ collection.id | json }},
        "name": {{ collection.title | json }},
        "products": [
          {%- for p in collection.products limit: 50 -%}
            {
              "id": {{ p.id | json }},
              "name": {{ p.title | json }},
              "price": {{ p.price | money_without_currency | json }},
              "position": {{ forloop.index }},
              "category": {{ p.type | json }}
            }{%- unless forloop.last -%},{%- endunless -%}
          {%- endfor -%}
        ]
      }
      {%- endif -%}
    }
  </script>
  ```

  Add module script load before `</body>`:
  ```liquid
  {%- if settings.ga4_measurement_id != blank -%}
    <script type="module" src="{{ 'ga4.js' | asset_url }}"></script>
  {%- endif -%}
  ```

  Verify: `grep -c "ga4_measurement_id" layout/theme.liquid` returns >= 2.

- [ ] **Task 2: Create assets/ga4.js — event helpers + influencer_code**

  Files: `assets/ga4.js`

  This ES module handles:

  1. **influencer_code dimension** — on DOMContentLoaded, read `utm_campaign` or a custom
     `ref` param from `location.search`. If found, store in `sessionStorage` under key
     `influencer_code`. On every page, read from sessionStorage and call:
     `gtag('set', { 'influencer_code': code })` so it attaches to all subsequent hits.

  2. **view_item** — if `ga4PageData.pageType === 'product'` and `ga4PageData.product`
     exists, push `gtag('event', 'view_item', { currency: 'USD', value: price, items: [item] })`.

  3. **view_item_list** — if `ga4PageData.pageType === 'collection'`, push
     `gtag('event', 'view_item_list', { item_list_id: collectionId, item_list_name: collectionName, items: [...] })`.

  4. **select_item** — attach click listeners to `[data-product-card]` elements (the class
     used by `snippets/product-card.liquid`). On click, push `gtag('event', 'select_item', ...)`.
     Use event delegation on `document` to handle AJAX-loaded cards.

  5. Export `trackAddToCart(cartItem)` and `trackBeginCheckout(cartData)` — called from
     pdp.js and cart-drawer.js respectively.

  Read page data from: `JSON.parse(document.getElementById('ga4-page-data').textContent)`.
  Guard all gtag calls with `typeof gtag !== 'undefined'` to prevent errors when GA4 is
  not loaded (e.g., ad blockers).

  Verify: `assets/ga4.js` exists; contains `view_item`, `view_item_list`, `trackAddToCart`,
  `trackBeginCheckout` strings.

- [ ] **Task 3: Wire add_to_cart and begin_checkout in pdp.js + cart-drawer.js**

  Files: `assets/pdp.js`, `assets/cart-drawer.js`

  In `assets/pdp.js`, after the Klaviyo `trackAddedToCart` call, add:
  - Import `trackAddToCart` from `./ga4.js`
  - Call `trackAddToCart(responseJson)` with the cart add response

  In `assets/cart-drawer.js`, on the checkout button click handler (the button that
  navigates to `/checkout`):
  - Import `trackBeginCheckout` from `./ga4.js`
  - Call `trackBeginCheckout(cartData)` where `cartData` is the `/cart.js` response already
    fetched by cart-drawer.js on cart updates

  For the `purchase` event: document in a Shopify Additional Scripts snippet (a plain
  `<script>` block, not a theme file) that the store owner must add to
  Settings → Checkout → Additional Scripts. The snippet text is documented in
  `06-CONTEXT.md` — it uses `{{ checkout.order_id }}`, `{{ checkout.total_price | money_without_currency }}`,
  and `{{ checkout.line_items }}` Liquid variables available only on the order status page.

  Verify: `grep -c "trackAddToCart" assets/pdp.js` returns >= 1; `grep -c "trackBeginCheckout" assets/cart-drawer.js` returns >= 1.
