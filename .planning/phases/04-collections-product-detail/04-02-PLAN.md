    Create assets/collection-filter.js as an ES module following the mobile-nav.js pattern (named exports + default init function called at bottom).

    MODULE STRUCTURE:
    - escapeHtml(str) helper: same implementation as in cart-drawer.js (& < > " ' replacements)
    - openFilterDrawer() / closeFilterDrawer() — toggle -translate-x-full on #filter-drawer, toggle hidden on #filter-drawer-overlay, lock/unlock document.body.style.overflow, apply focus trap (same getFocusableElements + trapFocus pattern from mobile-nav.js)
    - buildActiveChips(searchParams) — reads all filter params from searchParams, builds chip HTML, injects into #active-filter-chips. Each chip: `<span class="inline-flex items-center gap-1 bg-sand border border-deep/20 px-2 py-1 font-body text-xs text-deep">{{ label }}<button aria-label="Remove [label] filter" data-remove-param="{{ paramName }}" data-remove-value="{{ value }}" class="ml-1 text-mid hover:text-deep">×</button></span>`. Chip labels are derived from the param name by stripping `filter.` prefix and formatting.
    - fetchFilteredGrid(url) — async function: `const res = await fetch(url, {headers: {'X-Requested-With': 'XMLHttpRequest'}}); const text = await res.text(); const parser = new DOMParser(); const doc = parser.parseFromString(text, 'text/html'); const newGrid = doc.querySelector('#product-grid-inner'); const currentGrid = document.getElementById('product-grid-inner');` — if newGrid and currentGrid: `currentGrid.innerHTML = newGrid.innerHTML`. Also update product count: find `#product-count` in both old and new doc and swap text. If newGrid.children.length === 0: inject the no-match message `<div class="col-span-full py-24 text-center"><p class="font-body text-sm text-mid">No products match these filters. Try adjusting or clearing your filters.</p></div>` into currentGrid.
    - getFilterParams() — reads all checked `[data-filter-input]` checkboxes (name + value) and price range inputs ([data-filter-price-min] value, [data-filter-price-max] value) from the sidebar, returns a URLSearchParams object. Price params use keys `filter.v.price.gte` and `filter.v.price.lte`.
    - applyFilters() — calls getFilterParams(), appends current sort-by value as `sort_by`, appends `section_id=collection-grid`, builds URL as `window.location.pathname + '?' + params.toString()`, calls history.pushState({}, '', window.location.pathname + '?' + params.toString().replace('&section_id=collection-grid', '')), then calls fetchFilteredGrid(url).

    DEFAULT INIT:
    ```
    export default function init() {
      document.addEventListener('DOMContentLoaded', () => {
        // Drawer open/close
        document.getElementById('filter-drawer-open')?.addEventListener('click', openFilterDrawer);
        document.getElementById('filter-drawer-close')?.addEventListener('click', closeFilterDrawer);
        document.getElementById('filter-drawer-overlay')?.addEventListener('click', closeFilterDrawer);
        // "Show Results" in mobile drawer also closes drawer
        document.getElementById('filter-drawer-apply')?.addEventListener('click', () => { applyFilters(); closeFilterDrawer(); });
        // Escape key closes drawer
        document.addEventListener('keydown', (e) => { if (e.key === 'Escape') closeFilterDrawer(); });
        // Filter checkbox change (desktop sidebar)
        document.querySelectorAll('[data-filter-input]').forEach(input => {
          input.addEventListener('change', applyFilters);
        });
        // Sort change
        document.getElementById('sort-by')?.addEventListener('change', applyFilters);
        // Clear all
        document.getElementById('filter-clear-all')?.addEventListener('click', () => {
          document.querySelectorAll('[data-filter-input]:checked').forEach(cb => { cb.checked = false; });
          const minInput = document.querySelector('[data-filter-price-min]');
          const maxInput = document.querySelector('[data-filter-price-max]');
          if (minInput) minInput.value = '';
          if (maxInput) maxInput.value = '';
          applyFilters();
        });
        // Active chip removal (event delegation on chip container)
        document.getElementById('active-filter-chips')?.addEventListener('click', (e) => {
          const btn = e.target.closest('[data-remove-param]');
          if (!btn) return;
          const paramName = btn.dataset.removeParam;
          const paramValue = btn.dataset.removeValue;
          const input = document.querySelector(`[data-filter-input][name="${CSS.escape(paramName)}"][value="${CSS.escape(paramValue)}"]`);
          if (input) { input.checked = false; applyFilters(); }
        });
        // Hydrate active chips on page load (URL params already present if page was navigated to with filters)
        buildActiveChips(new URLSearchParams(window.location.search));
        // Sync sort-by select to current URL param
        const currentSort = new URLSearchParams(window.location.search).get('sort_by');
        const sortEl = document.getElementById('sort-by');
        if (sortEl && currentSort) sortEl.value = currentSort;
      });
    }
    init();
    ```

    SECURITY: All chip label text injected into innerHTML must be escaped through escapeHtml(). Param names and values from URL are never injected directly as HTML — only used as attribute values in chip button data attributes (escaped through template literals with escapeHtml). Variant IDs and quantity values from URL params are never used for cart operations in this module.

    CSRF: The fetch call to the collection URL uses `X-Requested-With: XMLHttpRequest` header (Shopify section rendering API pattern). No cart mutations occur in this module.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL params → AJAX fetch | Filter params from URL are constructed by the JS module, not from raw user input fields injected into HTML |
| Liquid output → browser | collection.title, filter.label, value.label, hero_subtitle are merchant-entered strings |
| Chip label → innerHTML | escapeHtml() must sanitize any label text before DOM injection |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-02-01 | Tampering | Liquid output of collection.title, filter labels | mitigate | Apply `| escape` on all Liquid string outputs in collection-grid.liquid |
| T-04-02-02 | Tampering | Active filter chip labels injected via innerHTML | mitigate | Pass all label strings through escapeHtml() before innerHTML injection in collection-filter.js |
| T-04-02-03 | Spoofing | AJAX fetch to collection URL | mitigate | Include `X-Requested-With: XMLHttpRequest` header on all fetch calls |
| T-04-02-04 | Tampering | data-remove-param / data-remove-value attribute values used in querySelector | mitigate | Use CSS.escape() when building querySelector string from data attribute values |
</threat_model>
