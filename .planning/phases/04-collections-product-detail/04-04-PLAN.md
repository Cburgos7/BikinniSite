## Trust Boundaries

|----------|-------------|
| pdp.js → /cart/add.js | variantId sourced from Shopify-rendered JSON — must validate as integer before use |
| Liquid output → browser | product.title, option values, metafield values are merchant-entered |
| product.description → innerHTML | Shopify rich text editor output; treated as sanitized by Shopify platform |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04-01 | Tampering | variantId sent to /cart/add.js | mitigate | Validate with parseInt(variantId, 10) and isFinite() check before fetch; reject NaN |
| T-04-04-02 | Tampering | Liquid output in product-main.liquid | mitigate | Apply `| escape` on product.title, option values, metafield string values; product.description exempt (Shopify-sanitized) |
| T-04-04-03 | Spoofing | Ajax POST to /cart/add.js | mitigate | Include `X-Requested-With: XMLHttpRequest` header on all cart fetch calls |
| T-04-04-04 | Denial of Service | Add-to-cart button double-submit | mitigate | Disable button during fetch; re-enable in finally block |
| T-04-04-05 | Elevation of Privilege | Lightbox dialog backdrop | accept | Native `<dialog>` element with showModal() — browser enforces focus trap and Escape handling natively |
</threat_model>
