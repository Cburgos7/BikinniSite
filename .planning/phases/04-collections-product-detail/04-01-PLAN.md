<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Shopify Liquid output → browser | Product title, option names, and aria-labels are merchant-entered and must be escaped before rendering |
| Wishlist button → /cart/add | Quick-add form submits variant ID from Liquid (server-side rendered — not user-controllable at this layer) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01-01 | Tampering | product.title in card markup | mitigate | Apply `| escape` filter on all product.title, option name, and aria-label outputs in product-card.liquid |
| T-04-01-02 | Information Disclosure | Wishlist toggle when customer not logged in | mitigate | Gate `data-wishlist-toggle` button behind `customer != blank`; unauthenticated users get `<a href="/account/login?return_url=...">` link only |
| T-04-01-03 | Tampering | Quick-add form variant ID | accept | Variant ID is rendered server-side from Shopify product object — not user-supplied at the card render layer |
</threat_model>
