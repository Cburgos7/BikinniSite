### Task 1.2 — package.json and PostCSS Build Pipeline

<action>
Create `package.json` with Tailwind CSS v3 + PostCSS build pipeline:

Dependencies to install:
- `tailwindcss@^3` 
- `postcss@^8`
- `autoprefixer@^10`
- `@shopify/prettier-plugin-liquid` (dev, for formatting)

Scripts:
- `"build": "postcss src/css/theme.css -o assets/theme.css"`
- `"watch": "postcss src/css/theme.css -o assets/theme.css --watch"`
- `"dev": "npm run watch"`

Create `postcss.config.js`:
```js
module.exports = {
  plugins: {
    tailwindcss: {},
    autoprefixer: {},
  },
}
```

Create `tailwind.config.js` with content paths set to all Liquid files:
```js
content: [
  './layout/**/*.liquid',
  './sections/**/*.liquid',
  './snippets/**/*.liquid',
  './templates/**/*.liquid',
  './templates/customers/**/*.liquid',
]
```

Theme object in `tailwind.config.js` extend.colors must include: `sand`, `deep`, `coral`, `gold`, `cream`, `mid` — values from buildSpec.md §1. Also set `fontFamily.display` to `['Cormorant Garamond', 'Georgia', 'serif']` and `fontFamily.body` to `['Barlow Condensed', 'Arial', 'sans-serif']`.
</action>

<acceptance_criteria>
- `npm install` runs without errors
- `npm run build` exits 0 and produces `assets/theme.css` containing Tailwind output
- `tailwind.config.js` content array includes all 4 Liquid directory globs
- `tailwind.config.js` extend.colors includes keys: sand, deep, coral, gold, cream, mid
- `postcss.config.js` lists `tailwindcss` and `autoprefixer` plugins
- `assets/theme.css` after build is non-empty (contains at least Tailwind reset CSS)
</acceptance_criteria>

### Task 1.3 — shopify.theme.toml and GitHub Auto-Deploy

<action>
Create `shopify.theme.toml` at repo root defining environments:

```toml
[environments.staging]
store = "YOUR_STORE_NAME.myshopify.com"
theme = "STAGING_THEME_ID"

[environments.production]
store = "YOUR_STORE_NAME.myshopify.com"
theme = "PRODUCTION_THEME_ID"
```

Use `[PLACEHOLDER]` for store name and theme IDs — these are filled in by the owner when connecting to Shopify.

Create `.github/workflows/shopify-deploy.yml` with two jobs:
1. `deploy-preview` — triggers on push to `staging` branch, runs `shopify theme push --environment staging --allow-live`
2. `deploy-live` — triggers on push to `main` branch, runs `shopify theme push --environment production --allow-live`

Both jobs use GitHub secrets: `SHOPIFY_CLI_THEME_TOKEN` and `SHOPIFY_STORE_URL`. Add a build step before push: `npm ci && npm run build`.

Add to `.gitignore`:
```
node_modules/
.DS_Store
*.log
.env
.env.local
```

Do NOT gitignore `assets/theme.css` — Shopify needs it committed for the theme push to work correctly.
</action>

<acceptance_criteria>
- `shopify.theme.toml` exists at repo root with `[environments.staging]` and `[environments.production]` sections
- `.github/workflows/shopify-deploy.yml` exists with two trigger conditions: `branches: [staging]` and `branches: [main]`
- Workflow YAML references `secrets.SHOPIFY_CLI_THEME_TOKEN` and `secrets.SHOPIFY_STORE_URL`
- Workflow includes `npm ci && npm run build` step before `shopify theme push`
- `.gitignore` contains `node_modules/` and `.env`
- `assets/theme.css` is NOT in `.gitignore`
</acceptance_criteria>
