## Tasks

### Task 3.1 — JS Utility Modules

<action>
Create the following ES module files in `assets/`:

**`assets/utils.js`** — shared utilities:
```js
// DOM query helpers
export const $ = (sel, ctx = document) => ctx.querySelector(sel);
export const $$ = (sel, ctx = document) => [...ctx.querySelectorAll(sel)];
export const ready = (fn) => {
  if (document.readyState !== 'loading') fn();
  else document.addEventListener('DOMContentLoaded', fn);
};
```

**`assets/intersection-observer.js`** — reusable reveal utility:
```js
export function observeReveal(elements, options = {}) {
  const defaults = { threshold: 0.15, rootMargin: '0px 0px -40px 0px' };
  const observer = new IntersectionObserver((entries) => {
    entries.forEach(entry => {
      if (entry.isIntersecting) {
        entry.target.classList.add('is-revealed');
        observer.unobserve(entry.target);
      }
    });
  }, { ...defaults, ...options });
  elements.forEach(el => observer.observe(el));
  return observer;
}
```

**`assets/custom-cursor.js`** — rAF-based custom cursor:
```js
let mouseX = 0, mouseY = 0;
let cursorX = 0, cursorY = 0;
const speed = 0.12;

export function initCustomCursor(cursorEl) {
  if (!cursorEl || window.matchMedia('(pointer: coarse)').matches) return;
  document.addEventListener('mousemove', e => {
    mouseX = e.clientX;
    mouseY = e.clientY;
  });
  function animate() {
    cursorX += (mouseX - cursorX) * speed;
    cursorY += (mouseY - cursorY) * speed;
    cursorEl.style.transform = `translate(${cursorX}px, ${cursorY}px)`;
    requestAnimationFrame(animate);
  }
  animate();
}
```

All files must use `export` (ES module syntax). No `import` of jQuery or any non-standard library. Each file is self-contained.
</action>

<acceptance_criteria>
- `assets/utils.js` exists and exports `$`, `$$`, `ready` functions
- `assets/intersection-observer.js` exists and exports `observeReveal` function
- `assets/custom-cursor.js` exists and exports `initCustomCursor` function
- No file contains `require(`, `import jQuery`, `$()` without the local alias, or any CDN script reference
- All three files use ES module `export` syntax
</acceptance_criteria>

### Task 3.2 — Wire JS to theme.liquid

<action>
Add the following to `layout/theme.liquid` in the `<body>`:

1. Custom cursor element (hidden on touch devices via CSS):
```html
<div id="custom-cursor" class="pointer-events-none fixed top-0 left-0 z-[9999] w-4 h-4 rounded-full bg-coral mix-blend-difference will-change-transform hidden md:block" aria-hidden="true"></div>
```

2. Module script at the end of `<body>`:
```html
<script type="module">
  import { initCustomCursor } from {{ 'custom-cursor.js' | asset_url | json }};
  const cursor = document.getElementById('custom-cursor');
  if (cursor) initCustomCursor(cursor);
</script>
```

Note: Shopify's `asset_url` filter returns a string — use the correct Liquid pattern for ES module imports. The standard approach is:
```liquid
<script type="module" src="{{ 'custom-cursor.js' | asset_url }}"></script>
```

And in `custom-cursor.js`, call `initCustomCursor` on `DOMContentLoaded` internally, or export and call from a separate entry module. Choose the simpler approach.

Also add `{{ content_for_layout }}` inside `<body>` where template content renders (Shopify requirement).
</action>

<acceptance_criteria>
- `layout/theme.liquid` contains `id="custom-cursor"` element in `<body>`
- `layout/theme.liquid` contains `{{ content_for_layout }}` in `<body>` (Shopify requirement)
- `assets/custom-cursor.js` is referenced via `<script type="module" src="{{ 'custom-cursor.js' | asset_url }}">` or equivalent
- No `<script src="https://code.jquery.com` or similar CDN references anywhere in layout/theme.liquid
</acceptance_criteria>
