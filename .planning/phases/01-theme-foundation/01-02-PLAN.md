<action>
Add a `:root` block to `src/css/theme.css` immediately after the `@tailwind base` directive:

```css
:root {
  --sand: #f5ede0;
  --deep: #0d0a08;
  --coral: #e85d3a;
  --gold: #c9a84c;

  --mid: #8a7060;
```

Verify `tailwind.config.js` extend.colors uses the exact same hex values (not `var(--sand)` references — Tailwind's JIT doesn't resolve CSS vars in color utilities by default). The extend.colors values set in PLAN-01 must use literal hex strings.

Run `npm run build` to confirm the CSS custom properties survive PostCSS output.
</action>

### Task 2.2 — Generate and Install WOFF2 Font Subsets

<action>
Generate subset WOFF2 files for:

**Cormorant Garamond:**
- `cormorant-garamond-regular.woff2` (weight 400, style normal)
- `cormorant-garamond-italic.woff2` (weight 400, style italic)
- `cormorant-garamond-semibold.woff2` (weight 600, style normal)

**Barlow Condensed:**
- `barlow-condensed-regular.woff2` (weight 400, style normal)
- `barlow-condensed-medium.woff2` (weight 500, style normal)

Subset to Latin character set (Unicode range U+0000-00FF, U+0131, U+0152-0153, U+02BB-02BC, U+02C6, U+02DA, U+02DC, U+2000-206F, U+2074, U+20AC, U+2122, U+2191, U+2193, U+2212, U+2215, U+FEFF, U+FFFD).

Use `pyftsubset` (fonttools) or equivalent. Source fonts from Google Fonts download or the font foundry. Place output files in `assets/`.

Add `@font-face` declarations to `src/css/theme.css` (after the `:root` block):

```css
@font-face {
  font-family: 'Cormorant Garamond';
  font-weight: 400;
  font-style: normal;
  font-display: swap;
  src: url('{{ "cormorant-garamond-regular.woff2" | asset_url }}') format('woff2');
}
/* repeat for italic, semibold */

@font-face {
  font-family: 'Barlow Condensed';
  font-weight: 400;
  font-style: normal;
  font-display: swap;
  src: url('{{ "barlow-condensed-regular.woff2" | asset_url }}') format('woff2');
}
/* repeat for medium */
```

Note: `@font-face` with Liquid `asset_url` filter works in Shopify because Liquid is rendered server-side before the CSS is served. This requires the CSS file to be processed as a Liquid asset (named `theme.css.liquid` or served through Shopify's asset pipeline). If using PostCSS output to `assets/theme.css`, embed the `asset_url` calls as literal CDN URLs instead — or use a `snippets/font-preload.liquid` approach where fonts are declared inline.

**Recommended approach:** Create `snippets/font-faces.liquid` with all `@font-face` declarations using Liquid `asset_url` filters, and render it in `layout/theme.liquid` within a `<style>` tag.
</action>

<acceptance_criteria>
- 5 WOFF2 files exist in `assets/`: cormorant-garamond-regular.woff2, cormorant-garamond-italic.woff2, cormorant-garamond-semibold.woff2, barlow-condensed-regular.woff2, barlow-condensed-medium.woff2
- Each WOFF2 file is < 60KB (properly subsetted)
- `@font-face` declarations exist for all 5 font files (either in src/css/theme.css, assets/theme.css, or snippets/font-faces.liquid)
- `font-display: swap` is present on all @font-face declarations
</acceptance_criteria>

### Task 2.3 — Font Preload Tags in theme.liquid

<action>
In `layout/theme.liquid`, add `<link rel="preload">` tags in the `<head>` for the 2 most critical font files (the ones loaded above the fold):

```liquid
<link rel="preload" as="font" type="font/woff2"
  href="{{ 'cormorant-garamond-regular.woff2' | asset_url }}"
  crossorigin="anonymous">
<link rel="preload" as="font" type="font/woff2"
  href="{{ 'barlow-condensed-regular.woff2' | asset_url }}"
  crossorigin="anonymous">
```

Place these preloads BEFORE the main CSS `<link>` tag. Preload only the 2 primary weights (regular) — do not preload italic/semibold/medium (they are not render-blocking).

Also render `{{ 'theme.css' | asset_url | stylesheet_tag }}` and the font-faces snippet if using that approach.

Full minimal `layout/theme.liquid` head should include in order:
1. `<meta charset="utf-8">` and viewport meta
2. `{{ page_title }}` title tag
3. Font preload tags (2 files)
4. Stylesheet link for theme.css
5. `{{ content_for_header }}` (Shopify requirement — must be present)
6. Font-face declarations (snippet or style tag)
</action>

<acceptance_criteria>
- `layout/theme.liquid` contains `<link rel="preload" as="font"` for cormorant-garamond-regular.woff2
- `layout/theme.liquid` contains `<link rel="preload" as="font"` for barlow-condensed-regular.woff2
- Both preload tags have `crossorigin="anonymous"` attribute
- `{{ content_for_header }}` is present in `layout/theme.liquid` (Shopify requirement)
- Font preload tags appear before the theme.css stylesheet link
- `{{ 'theme.css' | asset_url | stylesheet_tag }}` is present in theme.liquid
</acceptance_criteria>

## Verification

```bash
# Font files exist and are reasonably sized
for f in cormorant-garamond-regular cormorant-garamond-italic cormorant-garamond-semibold barlow-condensed-regular barlow-condensed-medium; do
  test -f "assets/${f}.woff2" && echo "PASS: ${f}.woff2" || echo "FAIL: ${f}.woff2 missing"
done

# CSS custom properties in built output
grep -c "var(--\|--sand\|--deep\|--coral" assets/theme.css && echo "PASS: tokens in CSS" || echo "FAIL"

# Tailwind tokens
grep -c "sand.*f5ede0\|deep.*0d0a08" tailwind.config.js && echo "PASS: Tailwind tokens" || echo "FAIL"

# Preload in layout
grep "rel=\"preload\"" layout/theme.liquid | wc -l | xargs -I{} sh -c 'test {} -ge 2 && echo "PASS: preloads present" || echo "FAIL: preloads missing"'
grep "content_for_header" layout/theme.liquid && echo "PASS: content_for_header present" || echo "FAIL"
```

## Must-Haves

- All 6 design tokens present as CSS custom properties in built CSS
- All 5 WOFF2 font files in assets/ with file size < 60KB each
- Font preload tags in theme.liquid for both primary font weights
- `content_for_header` present in theme.liquid (Shopify requirement — app scripts won't inject without it)
